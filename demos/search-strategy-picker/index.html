Hybrid Search

Combines BM25 keyword matching, semantic vector similarity, and graph context expansion into a single ranked result set.

When to use
  • General exploration -- you're not sure what you're looking for
  • Finding related code across naming conventions
  • Discovering both direct matches and conceptually similar entities
How it works
1. BM25Finds entities matching keywords (exact tokens)Fast
2. SemanticFinds entities with similar meaning (embeddings)Medium
3. GraphExpands results to include structurally related entitiesMedium
4. MergeDeduplicate, re-rank by combined scoreFast
Example
ie_query(project="agent-comm", query="message routing", mode="hybrid")
# Returns: Coordinator (0.015), close_connection (0.014), set_transport (0.013)
# Tags: bm25+semantic+graph

BM25 Keyword Search

Traditional full-text search using the BM25 ranking algorithm. Matches exact tokens in entity names, docstrings, and file paths.

When to use
  • You know the exact function or class name
  • Searching for specific identifiers or variable names
  • Need fastest possible results
Strengths & Weaknesses
Exact name matchExcellent
Partial name matchGood
Conceptual similarityPoor
SpeedFastest
Example
ie_query(project="agent-comm", query="HTTPTransport", mode="bm25")
# Returns exact matches for HTTPTransport class and related test functions

Semantic Vector Search

Uses all-MiniLM-L6-v2 embeddings to find entities by meaning rather than exact text. Queries and entities are compared in 384-dimensional vector space.

When to use
  • Conceptual queries: "error handling", "database connection"
  • Finding similar implementations with different naming
  • Exploring code you're unfamiliar with
Strengths & Weaknesses
Conceptual matchingExcellent
Cross-naming discoveryGood
Exact name matchFair
SpeedMedium
Example
ie_query(project="agent-comm", query="send data between agents", mode="semantic")
# Returns: Coordinator, MessageSpool, HTTPTransport (by meaning, not name)
